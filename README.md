## Что это

MSPro-Ltd Corp моделирует компанию как структуру отделений и должностей (постов) и наделяет каждый пост собственным AI-мозгом. В отличие от чат-ботов общего назначения, приложение знает оргструктуру конкретной компании, ведёт метрики каждого поста, автоматически оценивает его состояние по динамике показателей и помогает CEO-агенту принимать и **исполнять** управленческие решения — создавать посты, ставить задачи, накапливать опыт компании в долговременной памяти.

Работает полностью **локально**: данные хранятся в SQLite на машине владельца, секреты — под защитой Windows. AI — **двухконтурный**: облачный Claude как основной мозг и локальный Qwen 3 как резервный. Если облако недоступно, система продолжает работать.

## Возможности

- **Оргструктура** — отделения, отделы и посты с их ЦКП (ценными конечными продуктами) и метриками
- **Оргсхема** — визуальный конструктор структуры: отделение → отдел → агент, с карточкой агента (роль, мозг, MCP, ЦКП) и связями рабочего потока (кто кому передаёт результат, кто проверяет)
- **CEO-агент** — двухконтурный мозг: Claude с автоматическим fallback на локальный Qwen 3
- **Движок состояний** — метрики постов, автоматическая оценка состояния (от критического до образцового) по динамике показателей, тренды и спарклайны
- **База управленческих знаний** — встроенный справочник (модель оргструктуры, метрики, ЦКП, планирование), доступный CEO-агенту как инструмент
- **Интеллектуальный Диспетчер** — Hub-and-Spoke брокер задач: переписывает сырые запросы в развёрнутые промпты и маршрутизирует исполнителям
- **Посты-агенты** — пост может запустить собственный экземпляр AI в изолированном sandbox и произвести артефакт: `.docx` / `.xlsx` / `.pdf`
- **Память компании (Vault)** — файловое хранилище паттернов и решений, подмешивается в контекст CEO-агента
- **Security Vault** — секреты под Windows DPAPI (Credential Manager), без хранения в открытом виде
- **WebSocket-шлюз** — JSON-RPC 2.0 на `127.0.0.1:8899` с токен-авторизацией для внешних интеграций
- **COM-сервер** — `MSProLtdCorp.Application` для вызова из Python / VBA / PowerShell
- **Авто-обновление** — через GitHub Releases с подписанными MSI и откатом к предыдущей версии

## Технологический стек

| Слой | Технология |
|---|---|
| UI | React 19 + TypeScript 5.8 |
| Сборка фронтенда | Vite 7 |
| Бэкенд | Rust + Tauri v2 |
| База данных | SQLite (sqlx 0.8 + tauri-plugin-sql) |
| AI (основной) | Claude через локальный CLI |
| AI (резервный) | Qwen 3 локально (OpenAI-совместимый HTTP) |
| Секреты | Windows DPAPI (`keyring`) |
| Сеть | WebSocket (`tokio-tungstenite`) + `reqwest` |
| Интеграции | COM-сервер (`windows-implement`), Job Objects (`win32job`) |
| Обновления | `tauri-plugin-updater` (подпись minisign) |
| Платформа | Windows 10 / 11 (`.msi`) |

## Установка

### Для пользователей
1. Скачать последний `.msi` из [Releases](https://github.com/vladimirspecalp-hub/MSPro-Ltd-Corp-1.0/releases/latest)
2. Запустить установщик
3. Приложение само проверяет обновления и обновляется по кнопке в интерфейсе

### Для разработчиков
```bash
pnpm install
pnpm tauri dev    # разработка (фронтенд + бэкенд, hot-reload)
pnpm tauri build  # production-сборка .msi
```

Требования: **Node.js 20+**, **Rust 1.77+**, **pnpm**.
Перед коммитом: `pnpm tsc --noEmit` (0 ошибок) и `cd src-tauri && cargo test --lib`.

## Архитектура

```
┌──────────────────────────────────────────────┐
│  React 19 + TypeScript  (WebView2)            │
│  Главная · Чат CEO · Диспетчер · Оргсхема · ⚙ │
└───────────────┬──────────────────────────────┘
                │  Tauri invoke (40+ команд)
┌───────────────┴──────────────────────────────┐
│  Rust-ядро (Tauri v2)                         │
│  ├─ commands/         CEO chat, посты, диспетчер│
│  ├─ context_assembler динамическая сборка промпта│
│  ├─ vault             файловая память + знания  │
│  ├─ external_agent    WS-шлюз :8899 (JSON-RPC) │
│  ├─ com_server        MSProLtdCorp.Application  │
│  └─ secrets           DPAPI                     │
└───────┬────────────────────────┬───────────────┘
        │                        │
   ┌────┴─────┐          ┌───────┴────────┐
   │ SQLite   │          │ AI-мозги       │
   │ app.db   │          │ Claude / Qwen 3│
   └──────────┘          └────────────────┘
```

Данные живут в SQLite (`%APPDATA%\ru.msproltd.corp\app.db`), память компании — в файловом Vault рядом с БД. На каждый запрос CEO-агент собирает контекст (оргструктура + метрики + знания + история диалога с динамическим бюджетом окна модели) и обращается к мозгу. Ответ может содержать `tool_call`-блоки, которые ядро исполняет атомарно через SQLite.

## Roadmap

- **Оргсхема — материализация на диск**: сохранённый агент становится реальной рабочей папкой (роль, конфигурация MCP, память)
- **Оживление агентов**: запуск созданного агента через выбранный мозг и подключённые MCP-серверы
- **Конструктор-ассистент**: разворачивание целого отделения с потоком работы по одному запросу
- Суммаризация длинных диалогов и полнотекстовый поиск по истории
- MCP-мост для CEO-агента
